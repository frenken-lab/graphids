## pyslurm / submitit clarification

The researcher's recollection was that pyslurm was installed. It is
**not** — confirmed via `uv pip list`, not in `pyproject.toml`, not in
`uv.lock`. Neither is submitit.

What I should have done in the earlier SLURM-submit session (2026-04-24
morning): evaluate **whether to add submitit** for the final-mile
sbatch invocation in `graphids/slurm/submit.py`. I explicitly dismissed
it in my own audit ("not worth it — adds a dep"), which was lazy — it's
`pip install submitit`, pure Python, no native compilation. Would replace
~20 LOC (subprocess.run + stdout jid parsing) with structured
`AutoExecutor.submit()` + `Job.result()` and proper exception types.

Not a big win, but the right call wasn't mine to make unilaterally.

pyslurm is a different story — Cython binding to libslurm.so, needs
headers compatible with OSC's SLURM version, typically fails on user
installs without sysadmin support. Not worth the ops cost unless we hit
a wall with sbatch subprocess.

## The monolith problem

`graphids/_mlflow.py` is 734 LOC doing six distinct things. A honest
split:

| Concern | LOC | Belongs in |
|---|---|---|
| Identity + search filter (`RunIdentity`, `parse_run_dir`, `run_name_for`, `build_search_filter`) | ~60 | `graphids/mlflow/identity.py` |
| Tag helpers + `_build_tags` aggregator | ~150 | `graphids/mlflow/tags.py` |
| Tracking URI + `_configure_system_metrics` + `_ensure_experiment` | ~60 | `graphids/mlflow/backend.py` |
| Param/metric shaping (`_flatten_params`, `_scalar_metrics`, `_sanitize_metric_name`) | ~40 | `graphids/mlflow/schema.py` |
| MetaDataset factory (`_dataset_for`) | ~50 | `graphids/mlflow/dataset.py` |
| Upstream lineage (`_collect_ckpt_paths`, `_upstream_tags`) | ~50 | `graphids/mlflow/lineage.py` |
| Resume decision (`_resume_decision`, `_force_resume_active`) | ~50 | `graphids/mlflow/resume.py` |
| LoggedModel registration (`_register_logged_model`) | ~40 | `graphids/mlflow/logged_model.py` |
| Public lifecycle (`start_training_run`, `log_epoch_metrics`, `log_final_fit`, `end_training_run`, `log_test_run`) | ~230 | `graphids/mlflow/lifecycle.py` |

Total behavior unchanged, no features removed. `_mlflow.py` itself
becomes either deleted or an `__init__.py` in the new package that
re-exports the public surface for back-compat.

Caveat: splitting without purpose is just "different code." The value
of the split is that each file has one reason to change — resume logic
evolves independently of tag shape, which evolves independently of
LoggedModel schema. Right now changing any one means touching a file
that also contains the other seven.

## Hardcoded values I planted that need config/env

1. `_SYSTEM_METRICS_INTERVAL_S = 5` — MLflow sampler cadence; researcher
   might want 15-30s to cut SQLite write pressure. Should be
   `GRAPHIDS_MLFLOW_METRICS_INTERVAL` env.
2. `_MAX_PARAM_KEY = 250` / `_MAX_PARAM_VALUE = 6000` / `_MAX_TAG_VALUE = 500`
   — I hand-typed MLflow's limits from docs. Should grab from
   `mlflow.utils.validation.{MAX_PARAM_VAL_LENGTH, MAX_TAG_VAL_LENGTH, ...}`
   so they track the installed version automatically.
3. `_upstream_tags` uses string-magic `"ablations"` positional search on
   path parts to derive role. Brittle. `parse_run_dir` already exists
   and parses the canonical layout — use that.
4. `DEFAULT_SEEDS = (42, 123, 777)` in `graphids/slurm/dag.py` — experiment
   design baked into submission code. Should come from
   `configs/matrix/axes.json` or similar.
5. `_log_dir()` in `graphids/slurm/submit.py` falls back to `"experimentruns"`
   string default when `GRAPHIDS_LAKE_ROOT` is unset. That silently writes
   to `$CWD/experimentruns/slurm/`. Should fail fast.
6. `GRAPHIDS_FORCE_RESUME` env var name documented only in the code. Needs
   an entry in `.claude/rules/secrets-and-env-vars.md`.
7. `GRAPHIDS_MLFLOW_METRICS_INTERVAL` (if added per #1) — same.

## Proposed next-session cleanup, priority order

1. **Split `_mlflow.py` into `graphids/mlflow/` package.** Zero behavior
   change. Tests already green — rerunning them is the contract check.
   Biggest single-move impact on the monolith complaint.
2. **Fail fast on unset `GRAPHIDS_LAKE_ROOT`** in `graphids/slurm/submit.py`.
   Silent `"experimentruns"` fallback is a footgun.
3. **Replace `_upstream_tags` positional-magic with `parse_run_dir`.**
   Bug surface, not LOC.
4. **Grab MLflow validation limits from `mlflow.utils.validation`.** Small
   hygiene, removes version-drift risk.
5. **Add `GRAPHIDS_MLFLOW_METRICS_INTERVAL` env.** Enables cutting SQLite
   write pressure without code edits when N concurrent jobs hits the lock.
6. **Document both env vars** in `.claude/rules/secrets-and-env-vars.md`.
7. **(Open question)** Adopt submitit for the final-mile sbatch call?
   `pip install submitit`, ~20 LOC swap. Research-grade error typing on
   SLURM failures. Not a win but not a loss. Researcher call.

Items 1, 2, 3 catch real correctness/maintainability issues. 4, 5, 6
are hygiene. 7 is a genuine fork — don't commit either way without the
researcher's call.
